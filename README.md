# LEGO — список деталей по страницам инструкции

Веб-приложение для тех, кто собирает большие LEGO/MOC-модели по PDF-инструкции: указываешь диапазон страниц (например 250–400) — получаешь список всех уникальных деталей, которые нужны на этих страницах, с картинкой и итоговым количеством. Не нужно вручную пролистывать одни и те же страницы по разу на каждый цвет детали.

**Всё считается прямо в браузере.** PDF никуда не загружается и не отправляется на сервер — ни на какой. Это принципиально: сами инструкции обычно продаются с лицензией «только для личного использования», так что единственный правильный способ — обрабатывать файл локально, на устройстве пользователя.

Сайт — обычные статические файлы (`webapp/`), работает одинаково на Mac, Windows, Linux, а также в браузере телефона и планшета.

---

## 1. Быстрый старт — без установки

### Вариант А: открыть файл напрямую
Ничего устанавливать не нужно — ни Python, ни Node.js, ни библиотек. Нужен только браузер (Chrome, Edge, Safari или Firefox) и интернет при первом запуске (браузер один раз подгрузит библиотеку для чтения PDF с CDN).

- **Mac:** найдите папку `webapp` в Finder → дважды кликните `index.html` (откроется в браузере по умолчанию).
- **Windows:** найдите папку `webapp` в Проводнике → дважды кликните `index.html`.

### Вариант Б: одна ссылка для любого устройства (рекомендуется)
Если этот репозиторий выложен на GitHub, включите **GitHub Pages**, и сайт станет доступен по постоянной ссылке — можно открыть с телефона, не копируя никакие файлы:

1. На странице репозитория: **Settings → Pages**.
2. **Source** → `Deploy from a branch`, **Branch** → `main`, папка → `/webapp`, **Save**.
3. Через минуту-две GitHub покажет ссылку вида `https://<аккаунт>.github.io/<репозиторий>/` — сохраните её на телефоне.

PDF по-прежнему выбирается локально на устройстве и никуда не отправляется — сама страница просто раздаётся с GitHub, как обычный сайт.

## 2. Как пользоваться

1. Открыть сайт (см. выше).
2. Нажать «Выберите файл» и указать PDF-инструкцию.
3. Указать диапазон страниц (например, от 250 до 400).
4. Нажать «Собрать список деталей».
5. Через несколько секунд появится сетка карточек: картинка детали, суммарное количество (`×N`), страницы, на которых она встречается.

Если у какой-то детали количество помечено `?` — значит распознавание не до конца уверено в цифре, стоит перепроверить эту деталь глазами на нужной странице.

## 3. Как это работает (коротко)

Каждая страница инструкции сгенерирована стандартным конвейером (Stud.io/BrickLink) и имеет предсказуемую структуру: слева сверху — голубой блок со значками новых деталей для этого шага и подписями вида `2x`, `4x` под каждой.

1. [pdf.js](https://mozilla.github.io/pdf.js/) рисует страницу в `<canvas>`.
2. По цвету заливки (светло-голубой, `RGB 215,238,254`) находится сам блок — заливкой (flood fill) по связным областям, отдельно от других похожих по цвету «зум»-подсказок другого оттенка.
3. Блок нарезается на отдельные детали и текст количества по промежуткам фона (проекции по столбцам/строкам).
4. Количество читается **не через OCR**, а через собственный шаблонный распознаватель цифр: в `webapp/glyph-templates.js` зашиты эталонные изображения цифр `0–9` и `x`, вырезанные из реальных страниц этой же инструкции. Обычный OCR (Tesseract) в этом шрифте систематически путал «2» с «7» — сравнение с реальными эталонами оказалось надёжнее и в разы быстрее (не нужно грузить в браузер тяжёлый движок распознавания).
5. Иконки одинаковых деталей на разных страницах распознаются как одна и та же деталь по «отпечатку» изображения (перцептивный хеш + средний цвет), количества суммируются.

Проверено на реальном файле (692 страницы): на диапазоне в 150 страниц — 226 уникальных деталей, обработка меньше 15 секунд, только ~2% деталей помечены как «неуверенные».

## 4. Известные ограничения

- Шаблоны цифр вытащены из конкретного файла (`2.2-Super Yacht Deck 1 STERN 2.pdf`). Другие буклеты той же серии (того же автора/генератора) должны читаться так же хорошо — шрифт и стиль оформления у них общие. Если для другой инструкции (другой дизайнер/генератор) точность просядет — см. раздел про `tools/` ниже, шаблоны можно пересобрать под новый шрифт.
- Цифра «9» ни разу не встретилась в исходном диапазоне страниц, по которому строились шаблоны, — вместо неё используется приближение (шаблон «6», повёрнутый на 180°). Если в факте она распознаётся с ошибкой — можно доснять реальный шаблон «9» через `tools/` (см. ниже).
- Для очень старых/маломощных телефонов обработка большого диапазона (сотни страниц) может быть медленнее, чем на компьютере — вся работа идёт на устройстве пользователя.

## 5. Разработка: локальный сервер (по желанию)

Открыть `index.html` двойным кликом обычно достаточно. Но если браузер ругается на локальные файлы (`file://`) или вы правите код, удобнее поднять простой локальный сервер.

Оба варианта ниже требуют Python (на Mac он обычно уже есть; на Windows — поставить с [python.org](https://www.python.org/downloads/), при установке отметить галочку **Add python.exe to PATH**).

**Mac / Linux (Terminal):**
```bash
cd webapp
python3 -m http.server 8080
```
Открыть в браузере: `http://localhost:8080`

**Windows (PowerShell или cmd):**
```powershell
cd webapp
python -m http.server 8080
```
Открыть в браузере: `http://localhost:8080`

Чтобы открыть с телефона в той же Wi-Fi сети — вместо `localhost` укажите IP-адрес компьютера в локальной сети (Mac: `ipconfig getifaddr en0`; Windows: `ipconfig`, поле IPv4).

## 6. `tools/` — как пересобрать шаблоны цифр под другой файл/шрифт

Нужно только если точность распознавания количества упадёт на новом файле. Для обычного использования сайта эта папка не нужна.

### Установка (один раз)

**Mac:**
```bash
brew install tesseract          # нужен только для сверки через OCR внутри pdf_pipeline.py
cd tools
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**Windows (PowerShell):**
```powershell
# Tesseract (нужен только для сверки внутри pdf_pipeline.py):
# скачать установщик — https://github.com/UB-Mannheim/tesseract/wiki — и добавить путь в PATH
cd tools
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### Как пересобрать шаблоны

1. **Собрать образцы цифр из PDF:**
   ```bash
   python extract_glyph_templates.py "путь/к/инструкции.pdf" glyphs_out 5 300
   ```
   (диапазон страниц — любой достаточно большой кусок документа, чтобы встретились все цифры 0–9)

2. **Посмотреть, что получилось:** открыть `glyphs_out/cluster_00_nNNN.png`, `cluster_01_nNNN.png` и т.д. — каждый файл показывает одну уникальную форму символа, найденную в документе, отсортированные от самой частой к самой редкой.

3. **Прописать соответствие в `export_templates.py`:** отредактировать словарь `LABEL_MAP` в начале файла — какой номер кластера соответствует какой цифре (или `x`).

4. **Собрать итоговый файл:**
   ```bash
   python export_templates.py glyphs_out ../webapp/glyph-templates.js
   ```

`pdf_pipeline.py` — вспомогательный скрипт для проверки самого алгоритма (поиск блока, нарезка на детали, OCR-сверка) вне браузера, на реальных страницах:
```bash
python pdf_pipeline.py "путь/к/инструкции.pdf" out_dir 250 260
```
